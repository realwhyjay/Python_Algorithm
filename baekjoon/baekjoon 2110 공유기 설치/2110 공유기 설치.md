## ndb 클론 코딩

```python
n, m = map(int, input().split())
castle = []

# 성 상태를 입력받는다.
for _ in range(n):
    castle.append(input())

# 행과 열을 체크하기 위해 입력받은 n,m의 수만큼 0으로 채워진 리스트를 만든다.
row = [0] * n
colum = [0] * m

# 입력받은 성의 상태 중, 경비원의 위치를 파악한다.
for i in range(n):
    for j in range(m):
        if castle[i][j] == 'X':
            row[i] = 1
            colum[j] = 1

# 경비원이 없는 열을 체크한다
row_count = 0
for i in range(n):
    if row[i] == 0:
        row_count += 1

# 경비원이 없는 행을 체크한다
colum_count = 0
for i in range(m):
    if colum[i] == 0:
        colum_count += 1

# 경비원이 없는 열과 행의 수 중 더 큰 값을 출력한다.
print(max(row_count, colum_count))

```

## 다른 답 코딩

```python
import sys

N, C = map(int, input().split())
house = [int(sys.stdin.readline()) for _ in range(N)]

# 정렬
house.sort()

# 와이파이 간격을 얼마나 해야할지 모르니 최대거리와 최소거리의 중간부터 시작
start = 1
end = house[-1] - house[0]
answer = 0
while end >= start:
    # 중간값 (와이파이 간격)
    mid = (end + start) // 2

    # 와이파이 설치 # 처음 설치하고 시작
    count = 1
    value = house[0]
    for i in range(1, N):
        if house[i] >= value + mid:  # 설치 간격 충족
            count += 1
            value = house[i]

    # 와이파이 개수가 C보다 작으면 더 설치해야 한다. 간격을 좁히자.
    if count < C:
        end = mid - 1
    # 와이파이 개수가 C보다 많거나 같으면 덜 설치해야 한다. 간격을 넓히자.
    else:
        start = mid + 1
        # 중간에 mid를 저장하는 이유는 개수를 만족할 때 간격을 늘리려다가
        # 다음에는 개수를 만족시키지 못하고 while문이 종료될 수 있다.
        answer = mid

print(answer)
```

그대로 복붙해서, 변수만 위와 같게 수정해주었다.  
결과적으로 다른점을 찾자면 다음과 같다.

1. input()이 아닌 readline을 쓴다는 것.
2. house 에 append 를 해주는 방식. 
3. start 값을 house[1] - house[0] 이 아닌 1로 박아버린 것.

그럼 문제가 뭐냐???   
3번이 문제였다.   
앞서서 돌아가지 않던 모든 코드들에서 start를 1로 바꾸어주니 해결이 되었다.   
그렇지만 아직도 이해가 안되는건, 3번의 경우 오히려 다양한 반례들 (최소 거리가 1이 아닐 경우)에는 만족하지 못하는 거 아닌가..?
